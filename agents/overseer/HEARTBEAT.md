## 核心监控任务

### 1. 数据采集 [P0] ~10min
- [ ] **扫描会话**: 获取过去24小时所有Agent会话
  - 使用sessions_list获取活跃会话
  - 使用sessions_history获取详细历史
  - 覆盖所有Agent：wenyuan, shangqing, job-agent, forge, archen等
  
- [ ] **解析日志**: 提取关键指标
  - 调用次数统计
  - 工具使用频率
  - Token消耗估算
  - 执行时长记录
  - 成功/失败判定

- [ ] **数据清洗**: 处理异常记录
  - 跳过损坏的日志行
  - 处理并发会话冲突
  - 统一时间格式

### 2. 指标计算 [P0] ~5min
- [ ] **基础统计**: 各Agent调用次数、成功率
- [ ] **效能评分**: 按公式计算效能分
- [ ] **排名生成**: A/B/C等级分类
- [ ] **趋势对比**: 与昨日/上周对比

### 3. 洞察生成 [P1] ~5min
- [ ] **异常检测**: 识别低成功率Agent（<70%）
- [ ] **瓶颈识别**: 高Token消耗、长耗时任务
- [ ] **模式发现**: 高频工具使用、重复任务
- [ ] **优化建议**: 生成具体改进方向

### 4. 报告输出 [P0] ~5min
- [ ] **JSON报告**: metrics/daily/{date}.json
  - 结构化数据，供Architect消费
  - 包含完整统计数据和原始指标
  
- [ ] **Markdown报告**: metrics/reports/{date}.md
  - 可视化表格和图表
  - 人类可读的洞察和建议
  - 包含趋势对比

### 5. 告警通知 [P1] ~2min
- [ ] **检查P0告警**: 成功率<50%的Agent
  - 如有，立即发送message通知用户
  - 包含Agent名称、成功率、建议措施
  
- [ ] **检查P1告警**: 效能分连续下降
  - 在报告中标注
  - 汇总发送至用户

### 6. 数据维护 [P2] ~3min
- [ ] **归档旧数据**: 移动30天前的数据到archive
- [ ] **更新基线**: 重新计算7天滚动平均
- [ ] **清理缓存**: 删除临时文件

## 故障处理

| 故障场景 | 处理策略 |
|----------|----------|
| 数据采集失败 | 重试2次，如仍失败记录错误并跳过 |
| 指标计算异常 | 使用默认值，标记为估算 |
| 报告写入失败 | 尝试备用路径，通知用户 |
| 心跳超时 | 强制中断，下次执行时补全 |
