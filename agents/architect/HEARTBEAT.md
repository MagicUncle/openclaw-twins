## 前置检查

- [ ] 确认Overseer昨日报告已生成
  - 检查: `metrics/daily/{yesterday}.json`
  - 如不存在，延迟30分钟重试
  - 如仍不存在，记录警告并继续（使用历史数据）

## 核心分析任务

### 1. 状态加载 [P0] ~5min
- [ ] **读取报告**: 加载Overseer最新监控数据
  - JSON: metrics/daily/{yesterday}.json
  - 提取: Agent效能、成功率、排名、洞察
  
- [ ] **历史对比**: 加载近7天数据
  - 计算趋势变化
  - 识别退化Agent（连续下降）
  - 识别进步Agent

### 2. 缺口分析 [P0] ~10min
- [ ] **低性能Agent**
  - 筛选: 效能分 < 5 或 成功率 < 70%
  - 标记: 需要优化
  - 统计: 数量和分布
  
- [ ] **高成本Agent**
  - 筛选: Token消耗Top3
  - 分析: 成本结构和优化空间
  
- [ ] **能力缺口**
  - 检查: 常用工具覆盖度
  - 识别: 缺失的能力类型
  - 推测: 用户潜在需求
  
- [ ] **错误模式**
  - 聚类: 常见错误类型
  - 识别: 可修复的系统性问题

### 3. 知识获取 [P1] ~15min
对于每个识别的缺口：
- [ ] **Tavily搜索**
  - 查询: "{缺口描述} agent skill best practice"
  - 数量: 每个缺口3-5个结果
  - 过滤: 优先GitHub和技术博客
  
- [ ] **内容评估**
  - 相关性: 与缺口的匹配度
  - 可行性: 在当前系统的可实现性
  - 价值: 预期改进效果

### 4. 提案生成 [P0] ~15min
- [ ] **优化提案**
  - 针对: 低性能Agent
  - 内容: 问题描述、优化方案、参考链接
  - 分级: effort(低/中/高) × impact(低/中/高)
  
- [ ] **新技能提案**
  - 针对: 缺失的能力
  - 内容: 技能规格、使用场景、参考实现
  - 输出: 完整的SKILL.md草案
  
- [ ] **新Agent提案**（可选）
  - 针对: 系统性缺失或高频新需求
  - 条件: 仅当影响重大时生成
  - 内容: Agent角色定义、职责范围、技能需求

### 5. 提案整理 [P1] ~5min
- [ ] **JSON格式**: 保存结构化数据
  - 路径: `metrics/proposals/{date}/{id}.json`
  - 包含: 完整提案信息
  
- [ ] **Markdown格式**: 保存可读版本
  - 路径: `metrics/proposals/{date}/{id}.md`
  - 包含: 问题描述、方案详情、实施指南
  
- [ ] **汇总报告**: 当日所有提案
  - 路径: `metrics/proposals/{date}/summary.json`
  - 包含: 提案列表、优先级排序、推荐行动

### 6. 通知用户 [P0] ~5min
- [ ] **生成摘要**
  - 提案总数
  - 高优先级提案（P0/P1）
  - 推荐立即处理的提案
  
- [ ] **发送通知**
  - 方式: message工具
  - 内容: 摘要 + 查看详情链接
  - 行动: 等待人工确认

### 7. 学习沉淀 [P2] ~5min
- [ ] **保存搜索结果**
  - 整理: 有用的参考链接和摘要
  - 存储: `architect/data/knowledge/{topic}/`
  
- [ ] **更新学习笔记**
  - 记录: 今日学到的知识点
  - 路径: `architect/data/daily_notes/{date}.md`
  
- [ ] **提取规则**
  - 验证: 有效的优化模式
  - 保存: `architect/data/learned_rules/{category}.yaml`

## 故障处理

| 故障场景 | 处理策略 |
|----------|----------|
| Overseer报告缺失 | 等待30分钟重试，或使用历史数据 |
| Tavily搜索失败 | 记录错误，继续处理已有知识 |
| 提案生成超时 | 优先完成高优先级提案，其余延迟 |
| 通知发送失败 | 记录到日志，用户可手动检查提案 |
